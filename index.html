Definition of tags

  • #Severity Justification# - Document as why chosen severity is appropriate
  • #Problem Statement# - To document Problem Description
  • #Problem Verification# - To document how the problem was verified
  • #Question(s)# - To document Question(s)
  • #Root Cause# - To document root cause of the problem
  • #RCA Justification# - To document how this root cause resulted in problem
  • #Data Collected# - To document data collected and inferences
  • #Update# - To document non-technical / progress updates (with ETA)
  • #Customer Discussion#- To summarise WebEx / Phone sessions
  • #Action Plan# - To document action for customers (to share log, sample app etc.,)
  • #Solution# #Solution Plan# - To document resolution and step-by-step instructions
  • #Answer# - To document answer(s) with references
  • #Knowlede Content# - To refer KB article, if not WHY (both unpub)
